# 期权开仓信号捕捉系统 - 三层过滤器

**适用策略**：Covered Call / Cash-Secured Put / Wheel Strategy

**核心理念**：卖Put = 卖保险 → 保费高（IV高）+ 理赔概率低（市场稳定）= 最佳卖方环境

**系统类型**：Rule-Based（满足条件才开仓）

备兑看涨 (Covered Call) 和 现金担保看跌 (CSP) 本质上都是做空波动率 (Short Volatility) + 正向时间价值 (Positive Theta) 的策略，且带有轻微的多头倾向。
这两种策略最怕的是：标的暴跌（CSP 接飞刀，CC 亏正股）。
因此，你的三层过滤器设计的核心目标应该是：避开暴跌风险，寻找高波动率溢价。
---

## Layer 1: 市场过滤器

**核心问题：现在是卖期权的好时机吗？**

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | VIX 水平 | 15 ~ 30 | CBOE波动率指数，衡量S&P500未来30天隐含波动率 | 15-30保费足够且市场未恐慌；<15保费太低不值得；>30风险加大需谨慎；>40暂停 |
| P1 🔴 | VIX期限结构 | VIX/VIX3M < 1.0 | VIX(30天) / VIX3M(3个月) | <1.0正向结构正常；>1.0反向=近期有黑天鹅预期；>1.1暂停 |
| P1 🔴 | 宏观事件 | 无重大事件 | FOMC/CPI/非农等发布前3天 | 二元事件前不确定性太大，等事件过后 |
| P2 | VIX Percentile | > 30% | (当前VIX在过去252个交易日中的百分位排名) | 50%-80%最佳，表示保费处于历史较高位 |
| P3 | SPY趋势 | SPY > MA50 | SPY收盘价 vs 50日简单移动平均线 | 确保处于技术性牛市或震荡市，行权概率更小 |
| P3 | ADX趋势强度 | < 30 | Average Directional Index，衡量趋势强度（非方向） | <25震荡市=卖方最爱；>40强趋势不适合卖方 |
| P4 | Put/Call Ratio | > 1.0 | CBOE股票期权Put成交量/Call成交量 | >1.0市场悲观=逆向信号=底部概率大 |

## Layer 2: 标的过滤器

**核心问题：这个标的适合卖期权吗？我愿意长期持有吗？**

> 注意：期权合约的流动性指标（期权成交量、Bid-Ask Spread、Open Interest）已移至 Layer 3 合约过滤器。
> Layer 2 只检查标的本身的条件。

### 2.1 标的流动性指标

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P2 | 股票成交量 | > 500K股 | 标的股票当日(已开盘)或昨日(未开盘)成交量 | 标的本身需有足够流动性 |

### 2.2 波动率指标

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | IV / HV | > 1.0 | 30日IV / 30日HV（历史波动率） | >1.0=卖"贵"的波动率；<0.9暂停=卖"便宜"不划算 |
| P2 | IV Rank | > 30% | (当前IV - 52周最低IV) / (52周最高IV - 52周最低IV) | >50%理想，表示IV处于历史高位 |
| P3 | IV Percentile | > 30% | 过去252天中IV低于当前的天数占比 | 与IV Rank互相验证 |

### 2.3 基本面指标

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P2 | 市值 | > $10B | 股价 × 总股本 | 大盘股(>$10B)更稳定，小盘股(<$2B)波动大 |
| P3 | PE vs 历史 | 不在历史高位 | 当前PE在过去5年PE范围中的位置 | 历史低位=悲观已定价；历史高位=期望满失望空间大 |
| P3 | 分析师评级 | Hold ~ Buy | 各大机构评级汇总（1-5分制，1=Strong Buy） | 2.5-3.5最佳；Strong Buy期望太高反而危险（逆向思维） |

### 2.4 技术面指标

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P2 | RSI | 35 ~ 65 | RSI(14) = 100 - 100/(1 + 平均涨幅/平均跌幅) | 40-60健康；<30超卖可能继续跌；>70超买可能回调 |
| P2 | 价格vs MA200 | 价格 > MA200 | 收盘价 vs 200日简单移动平均线 | 长期上升趋势，适合卖Put |
| P3 | Support Distance (CSP) | > 5% | (当前价 - 最近支撑位) / 当前价 × 100% | 安全垫厚=Strike设在支撑下方有缓冲 |
| P3 | Resistance Distance (CC) | 阻力位明确 | 识别前高、密集成交区等技术阻力 | Strike设在阻力上方，不易被Call走 |

### 2.5 事件日历

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | 财报日期 | > 7天 或 合约在财报前到期 | 距下次财报发布的自然日 | 财报=二元事件，跨财报持仓风险不可控 |
| P1 🔴 | 重大事件 | 无 | FDA审批、并购、诉讼裁决、拆股等 | 二元事件结果不可预测 |
| P2 | 除息日 (CC) | > 7天 | 距下次除息日的自然日 | <7天的ITM Call可能被提前行权 |

## Layer 3: 合约过滤器

**核心问题：选择哪个Strike和到期日？**
**过滤顺序：DTE → Strike → 收益风险验证**

### 3.1 DTE选择

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | DTE | 14 ~ 45天 | Days to Expiration，距到期自然日 | Theta∝1/√T加速衰减；Gamma∝1/T风险可控；<7天Gamma爆炸 |
| P2 | 财报跨越 | 不跨财报 | 合约到期日 vs 财报日期 | 跨财报=赌博 |

### 3.2 Strike选择

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | \|Delta\| | 0.10 ~ 0.35 | 期权Delta绝对值，来自期权链 | 0.10≈90%胜率；>0.45太近ATM风险大 |
| P2 | OTM % | 5% ~ 15% | Put: (S-K)/S; Call: (K-S)/S | 足够安全边际但权利金不会太少 |
| P2 | Strike位置 | 在支撑/阻力位之外 | CSP: K<支撑; CC: K>阻力 | 利用技术位作为额外保护 |
| P3 | Win Probability | > 65% | ≈ 1 - \|Delta\| 或 精确=N(d2) | 胜率基本保障 |

### 3.3 收益风险验证

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P0 🔴🔴 | Expected ROC | > 10% | E[ROC] = p×MaxProfit - (1-p)×E[Loss]，年化 | 期望收益需足够高才值得做 |
| P1 🔴 | TGR | > 0.5 | \|Theta\| / (\|Gamma\| × S² × σ_daily) × 100 | 标准化时间收益/波动风险比 |
| P1 🔴 | 费率 | > 1% | Premium / K × 100% | 类似保险公司保费/赔付金比率 |
| P1 🔴 | 年化夏普比率 | > 0.5 | SR_annual = SR × √(365/DTE)，其中 SR = (E[R]-Rf)/Std | 年化风险调整收益，反映策略可持续性 |
| P2 | ROC (年化) | > 15% | Premium / Margin × (365/DTE) | 最大收益率，需>15%才值得做 |
| P3 | Theta/Premium | > 1%/天 | \|Theta\| / Premium | 每天赚取权利金的效率 |
| P3 | Kellt Fraction | > 1% |f_kelly = E[π] /Var[π]| 仓位控制推荐，只用1/4作为实际仓位上限。 |

### 3.4 合约流动性

| 优先级 | 指标 | 开仓条件 | 定义/公式 | 逻辑解释 |
|--------|------|----------|-----------|----------|
| P1 🔴 | Bid-Ask Spread | < 5% | (Ask - Bid) / Mid × 100%，目标合约 | >5%摩擦成本吞噬收益 |
| P1 🔴 | Open Interest | > 500 | 目标Strike的未平仓合约数 | 低OI可能无法及时平仓 |
| P3 | Volume Today | > 50 | 当日成交量 | 有交易活跃度 |

## 优先级说明

| 优先级 | 含义 | 处理方式 |
|--------|------|----------|
| P0 🔴🔴 | 致命条件 | 不满足=立即排除，无例外 |
| P1 🔴 | 核心条件 | 不满足=强烈建议不开仓 |
| P2 | 重要条件 | 不满足=需要其他条件补偿或减少仓位 |
| P3 | 参考条件 | 不满足=可接受，但需知悉风险 |
| P4 | 辅助条件 | 锦上添花，非必须 |

## 开仓决策流程

```
Step 1: Layer 1 市场检查
        └─→ 任何P0/P1红灯 → 当日暂停所有新开仓

Step 2: Layer 2 标的筛选
        └─→ 波动率数据缺失(P1) → 该标的排除
        └─→ IV/HV不在范围内(P1) → 该标的排除
        └─→ 财报日期<7天(P1) → 该标的排除
        └─→ 其他P2/P3条件 → 警告但不排除，生成候选列表

Step 3: Layer 3 合约选择
        └─→ 期权流动性P0不满足 → 该合约排除
        └─→ Expected ROC < 0%(P0) → 该合约排除
        └─→ TGR/夏普P1不满足 → 该合约排除
        └─→ Open Interest不足(P1) → 该合约排除
        └─→ 通过所有检查 → 进入仓位决策

Step 4: 仓位决策
        └─→ Kelly f* = (p×b - q) / b
        └─→ 实际仓位 = 1/4 ~ 1/2 Kelly
        └─→ 单标的不超过总仓位5%
```

## 关键公式速查

| 公式 | 说明 |
|------|------|
| Win Prob ≈ 1 - \|Delta\| | 胜率简化估算 |
| Win Prob = N(d2) | 精确胜率，d2 = [ln(S/K) + (r - σ²/2)×T] / (σ×√T) |
| IV Rank = (IV - IV_min) / (IV_max - IV_min) | 当前IV在52周范围的位置 |
| ROC = Premium / Margin × (365/DTE) | 年化收益率 |
| E[ROC] = p × MaxProfit - (1-p) × E[Loss] | 期望收益率 |
| SR_annual = SR × √(365/DTE) | 年化夏普比率，其中 SR = (E[R]-Rf)/Std |
| 费率 = Premium / K × 100% | 期权费率，类似保险保费/赔付金 |
| TGR = \|Theta\| / (\|Gamma\| × S² × σ_daily) × 100 | 标准化Theta/Gamma比，Gamma Dollar (Γ × S²) 消除价格影响 |
| Kelly f* = (p × b - q) / b | 最优仓位比例，p=胜率，b=赔率，q=1-p |
| Theta ∝ 1/√T | 时间越短，Theta衰减越快 |
| Gamma ∝ 1/T | 时间越短，Gamma风险越大 |
| RSI = 100 - 100/(1 + AvgGain/AvgLoss) | 相对强弱指数 |
| ADX = MA(\|+DI - -DI\| / (+DI + -DI)) × 100 | 趋势强度指数 |

## 常见二元事件清单

### 宏观市场级别
- FOMC利率决议
- 非农就业数据
- CPI/PPI通胀数据
- GDP数据
- 总统大选/中期选举
- 地缘政治（战争、制裁、关税）

### 标的级别
- 财报发布（Earnings）
- FDA药物审批（生物科技致命事件）
- 并购/收购消息
- 重大诉讼裁决
- 拆股/合股公告
- CEO/管理层突然变动
- 信用评级变化
- 监管处罚/反垄断调查结果

**二元事件特点**：结果只有两种可能（好/坏），事件前IV膨胀，事件后IV Crush。卖方跨事件持仓=赌博。
